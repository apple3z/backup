## 一、核心理念（两条铁律）

1. **"无文档，不开发"**
   - 所有研发活动必须以文档为载体
   - 文档即代码，同步更新

2. **"Web端是唯一的协作界面"**
   - 所有文档通过Web端查看/编辑
   - MD文档是唯一的协作载体

## 三、研发流程（三阶段循环）

### Phase 1: 需求协同阶段
```
① 张总提出目标和需求
② 瑞贝卡需求分析
③ 开启新版本
④ 编写需求文档 (02-需求.md)
⑤ Web端协同修改
⑥ 需求确认签字 ✅
```

### Phase 2: 研发实施阶段
```
① 瑞贝卡阅读《研发管理规范.md》
② 架构设计 → 技术设计 → 编码实现
③ 测试计划 → 测试执行 → 问题修复
④ 全程更新文档（09-迭代日志.md, 11-状态看板.md）
⑤ 研发完成，待体验
```

### Phase 3: 体验反馈阶段
```
① 张总实际使用体验
② 发现问题记录 (08-问题清单.md)
③ 循环修复直到无重大问题
④ 版本研发完毕
```

### Release: 正式发布
```
① 获取Release版本
② 版本正式发布
③ 本次研发结束
```

## 六、协同规范

1. ✅ 阅读《研发管理规范.md》
