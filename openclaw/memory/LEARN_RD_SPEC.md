# 📖 研发规范学习笔记

> 学习时间: 2026-03-07

---

## 核心原则

1. **无文档，不开发** - 所有研发活动必须以文档为载体
2. **Web端是唯一的协作界面** - 所有文档通过Web端查看/编辑
3. **文档驱动研发** - 文档即代码，同步更新

---

## 项目结构

### 1. 系统管理 (协同办公)
- **版本管理/** - 每个版本的12个标准文档
- **研发规范/** - 现役规范文档

### 2. 业务功能
- 新闻系统
- Skills管理
- 订阅管理
- Wiki文档

---

## 研发流程 (三大阶段循环)

### Phase 1: 需求协同阶段
1. 张总提出需求
2. 瑞贝卡分析需求
3. 编写需求文档 (02-需求.md)
4. Web端协同修改
5. 需求确认签字 ✅

### Phase 2: 研发实施阶段
1. 瑞贝卡阅读规范
2. 架构设计 (03-架构.md)
3. 技术设计 (04-技术.md)
4. 编码实现
5. 测试计划 (06-测试计划.md)
6. 测试执行 (07-测试报告.md)
7. 问题修复 (08-问题清单.md)
8. 每日更新 09-迭代日志.md, 11-状态看板.md

### Phase 3: 体验反馈阶段
1. 张总体验
2. 发现问题记录
3. 循环修复直到无重大问题

### Release: 正式发布

---

## 每次开启项目前必须

1. ✅ 阅读《研发管理规范.md》
2. ✅ 确认当前处于哪个阶段
3. ✅ 明确需要输出哪些文档
